## Разработка DAG

#### Создание DAG:
Напишите DAG в соответствии с документацией Apache Airflow. Укажите необходимые параметры (начальная дата, расписание, задачи, зависимости).
Используйте PEP8 и форматирование с помощью Black.


#### 1. Локальное тестирование

Перед отправкой изменений убедитесь, что ваш DAG корректно загружается и не содержит синтаксических ошибок:
###### Тестирование DAG:
Если у вас настроено локальное окружение Airflow или установлен pytest, выполните:
```bash
pytest tests/
```
Тесты проверяют импорт и загрузку всех DAG в каталоге.

###### Линтинг и форматирование:
Запустите команды для проверки стиля:
```bash
black --check .
flake8 .
```
Если обнаружены проблемы, исправьте их, запустив:
```bash
black .
```
##### Процесс развертывания (CI/CD)
1. Проверяет стиль и тестирует DAG.
2. Копирует обновлённые файлы из каталога `/dags` в директорию `/opt/airflow/dags/` на сервере.
3. Если контейнер Airflow не запущен, онзапускается с помощью Docker Compose.
Вам достаточно обновить или добавить новые DAG-файлы в папку `dags/` и создать Merge Request. После успешного прохождения тестов и линтинга новые DAG появятся в Airflow автоматически.

##### Работа через веб-интерфейс Airflow
###### Доступ к Airflow:
После развёртывания откройте веб-интерфейс Airflow (обычно доступен по адресу, указанному администратором, например, http://localhost:8080).
###### Запуск DAG:
Используйте вкладки “Graph View”, “Tree View” и “Task Instance Details” для отслеживания выполнения ваших задач. При возникновении ошибок изучите логи для диагностики.
